# KK Quiz

KK Quiz — универсальный модуль квизов для 1C-Битрикс.

## Технические требования

- PHP 8.2+
- JavaScript ES6+
- 1C-Битрикс с поддержкой D7
- Модуль iblock

## Архитектура

- 1 инфоблок для структуры квизов
- 1 инфоблок для заявок
- раздел инфоблока = отдельный квиз
- элемент QUESTION = вопрос
- элемент RESULT = результат
- ответы вопроса будут храниться через кастомное свойство KK_ANSWERS

## Стандарт разработки

- PHP 8.2+
- JavaScript ES6+
- 1C-Битрикс D7
- Основная бизнес-логика должна быть в D7-стиле:
  - namespace Kk\Quiz
  - autoload через Loader::registerAutoLoadClasses
  - сервисы в /lib/Service
  - репозитории в /lib/Repository
  - AJAX-контроллеры через Bitrix\Main\Engine\Controller
- Legacy API допускается только изолированно:
  - установка инфоблоков
  - создание свойств
  - создание пользовательских полей
  - операции, где D7 API неполный или неудобный
- Компоненты и фронтенд не должны напрямую работать с CIBlockElement/CIBlockSection.
  Доступ к данным должен идти через Repository-классы.

## Свойство KK_ANSWERS

KK_ANSWERS — кастомное свойство инфоблока для хранения вариантов ответа вопроса.
Редактируется в админке как карточки ответов, внутри хранится технический JSON.
Связанные вопросы и результаты выбираются из текущего раздела-квиза, а для картинки ответа показывается превью по ID файла.
Для варианта ответа можно загрузить изображение. В JSON хранится ID файла Битрикса в поле image_id.
В админке элементы инфоблока kk_quizzes используют динамический вывод свойств:
- для QUESTION показываются свойства вопроса;
- для RESULT показываются свойства результата;
- общие свойства показываются всегда.

## Коды сущностей

- код квиза = стандартное поле CODE раздела инфоблока;
- код вопроса = стандартное поле CODE элемента QUESTION;
- код результата = стандартное поле CODE элемента RESULT.

Дополнительное свойство KK_CODE не используется.

## Чтение квиза

Для получения публичной структуры квиза используется:

```php
\Kk\Quiz\Service\QuizService::getPublicQuiz(string $code): ?array
```

Код квиза — это стандартное поле CODE раздела инфоблока.
Коды вопросов и результатов — это стандартное поле CODE элементов.

## Публичный компонент

Первый публичный вывод квиза доступен через компонент `kk:quiz`:

```php
<?$APPLICATION->IncludeComponent(
    "kk:quiz",
    "",
    [
        "QUIZ_CODE" => "pc_selector",
        "DISPLAY_MODE" => "block",
    ]
);?>
```

На первом этапе компонент выводит квиз и финальную форму-заглушку. Сохранение заявки будет реализовано отдельным этапом.

### Оформление и изображения ответов

В настройках раздела-квиза доступны необязательные тема (`light`/`dark`), основной, hover, active и progress
цвета в HEX, отдельные скругления контейнера, карточек, кнопок, полей и изображений, глобальное соотношение
сторон изображений ответов и режим `cover`/`contain`. Поддерживаются `1:1`, `3:4`, `4:3`, `9:16` и `16:9`.

Свойства вопроса «Соотношение сторон изображений ответов» и «Режим отображения изображений ответов»
переопределяют глобальные значения; вариант «Как у квиза» сохраняет наследование. Старое общее поле
скругления продолжает использоваться как fallback. Если визуальные настройки не заполнены, применяются
безопасные значения: светлая тема, синий акцент, скругления 24/16/12/10/12 px, `4:3` и `cover`.
